文件上传漏洞

Arbitrary File Upload，任意文件上传是指用户上传了一个可执行的脚本文件，并且此脚本文件获得了执行服务端命令的能力

原因：服务器对上传文件处理逻辑不够安全

一句话木马

漏洞情况

web server端设计缺陷——路径解析问题
  • Apache 对后缀解析从右向左，如果后缀不在已知后缀名单中，向左继续寻找，如test.php.a.b
  • php cgi 解析 http://www.a.com/test.jpg/notexist.php 此时如果这个notexist.php文件不存在，会将test.jpg当做php处理
  • IIS 6 下，当文件名为abc.php;xx.jpg 时，会将其解析为abc.php
——.htaccess文件
  • 前提，Apache需要开启两个设置，mod_rewrite ，AllowOverride All
  • 在允许上传.htaccess的场合，上传该文件可以让Apache按照.htaccess文件中的规则执行解析方案

抓包，content-disposition中改filename=”.htaccess” ,在http正文头添加AddType=application/x-httpd-php .jpg 意思是把jpg当做php格式解析

前端检查

修改js，或者抓包修改，绕过

content-type绕过

抓包绕过，修改包中的content-type字段

黑名单绕过

过滤php关键词的话，可以把代码放在html中当做php执行，或者修改为phtml php3 php4 php5 Php php (空格) php

双写、大小写绕过

pphphp，有的只过滤小写php，可以写成大写PHP一样执行

%00截断
条件竞争
.绕过 ::$DATA绕过(WINDOWS)

绕过思路

前端js绕过

1、如果是前端弹窗，大概率是js验证，f12修改js，如果遇到js混淆，正则表达式替换法用在线工具，基于DOM文档树的混淆考虑调试函数功能

2、抓包修改

黑白名单

尝试pphphp这样的后缀，如果上传成功，尝试能否被解析成功

后端对http包的检查

提示：文件类型不正确：抓包修改http头部content-type

如：application/octet-stream换成image/jpeg

还会对内容，文件头检查

文件头绕过：在http内容首行添加GIF89a,用图片格式头冒充php文件

内容绕过：先上传图片格式木马，再修改回php格式

路径截断绕过

00截断原理：0x00和%00经过解析后会变成chr(0)，0是ASCII中的null对应空格，从而把文件名中用于伪装传输格式的后缀名在前端解析时截断。

0x00对应在hex中修改的空格，**%00在url中，如xx.php?filename=test.php%00.txt** 或者 抓包中 path=shell.jsp%00.txt

常见于asp，php<5.3.4版本，jsp也会有，前端绕过可能有效，后端无效

filename.php.0x00.jpg Bp抓包，先在要截断的地方输入在hex进制好找的，比如空格对应hex的20，找到，改成00

前端绕过了，但是后端还会把从第一个点开始，验证后面的后缀是否等于指定的几种格式，这时即使绕过前端验证，后端也会被拦截，这时如果抓包发现有上传路径，可以把空格先放在路径中等着上传文件。

另外一种情况，有时候会用文件名拼接路径形成真正的上传路径，此时如果没有上面的后缀检查，文件名中含有空格的后缀还是可以被一起拼接到新的路径中的，空格依然生效。

空格放上传路径，坐等文件拼接， 如果把空格放在上传文件名中过不了后端的安检，但是抓包中发现了上传路径，可以修改包，在路径后面添加aa.php%00 原理是，当经过安检后，到了最后文件拼接时，会被拼接形成aa.php%00/filename.jpg 此时空格生效，相当于还是把文件里面木马放到空路径aa.php中了，参考php代码如下

pikachu靶场——文件相关

文件包含

​ 文件包含漏洞，会把文本格式的东西或者其他东西，都当作php代码来执行

    本地文件包含
        <?php
        $pifi=$_GET['pifu'];
        include($pifu.'php');
        echo 'url中传入pifi=pifu1或pifi2分别对应后台pifi1.php和pifu2.php，动态加载文件会使得当前页面内容简洁'
        ?>

        filename=../abcmuma.txt   include()中支持../ 也可以包含上一级
        此时abcmuma.txt文件中是php代码<?php phpinfo() ?>  虽然txt不能直接被当做php，但文件包含时，该txt被放在include()中会被当做php执行

        安全写法:白名单写死了，严格指定包含文件
        if(&filename=='file1.php'||$filename=='file2.php'||$filename='file3.php') 


    远程文件包含
        phpstudy-设置-配置文件-5.6.9版本-php.ini 
        allow_url_include如果是On，允许远程文件包含，重启服务
        filename=http://127.0.0.1:8011/abcmuma.txt
        防：ini配置文件中操作

文件上传

    <?php eval($_POST['haha'])?> 一句话木马，传上去之后，url参数haha=phpinfo()  eval会把识别为php

    前端拦截

    windows如果有的时候无法成功

    也可以使用kali！！！！
    cat 1.jpg muma.php > 2.jpg

命令执行漏洞

​ php里面使用了命令执行函数
​ shell_exec exec system popen proc_popen pcntl_exec passthru

phpstudy开启网站，index.php  

<?php 
    $ip=$_GET['ip'];  //实现从url中传入需要ping的目标 【】中的就是url中的参数名称ip=xxx
    shell_exec('ping'.$ip)  //网站的php文件中有命令执行漏洞
    $result=shell_exec('ping www.baidu.com');
    echo $result;
?>

|   左边运行结果作为右边的输入
||  qr前面执行不成功就执行右边，左边运行成功就不运行右边
&   先运行左边后运行右边
&&  左边运行完再运行右边
;   linux中特有，左边执行完继续右边

防御：上面符号替换为空

ping 127.0.0.1 && dir
whoami 查看我是谁
ipconfig 查看ip地址（Windows，Linux使用ifconfig）
net user zj 123 /add 增加用户

eval(字符串)
eval可以把字符串当作php代码来执行！！！

eval  vs  代码执行函数： eval能让php代码执行函数起作用，php代码执行函数能让dir这种系统命令起作用，eval没法直接让dir起作用
如果黑客利用了eval，在里面传入上面php代码执行的函数，那你就死了
