挖矿应急

Created2024-03-19|Updated2024-03-20|折腾吧
|Word count:1.1k|Reading time:4min|Post View:

网络连接处理

netstat -anop | grep ESTABLISHED

iptables封禁ip

rcu_shader连接是没了（见用户文件处理部分，rcu这个文件属于lenovo用户下，直接userdel了），todesk和那个-矿池还在

删除todeskd服务（见后门处理部分），还剩-矿池，有意思的是禁了一个就又换了一个IP地址

根据端口查看进程和相关文件，还是没查到，

netstat -tunap

结果发现smbd，135，445端口在监听着，不需要和win之间共享，关闭systemctl stop/disable smbd/nmbd 这两个都属于samba服务，samba用于和win共享打印机和文件服务，关闭avahi-daemon服务（Linux系统中的一个服务，它实现了Zeroconf网络协议，也称为零配置网络。这项服务允许设备在没有DNS服务器的情况下自动发现网络上的其他设备和服务，如打印机和文件服务器。）

netstat低版本有bug，Linux分配的进程连接的socket id会逐渐增大。当服务器运行较长的一段时间，期间没有进行断电重启时，这个socket id将会超过 INT_MAX （即2^31）。而netstat的源代码中对socket id大于INT_MAX的进程不予处理。所以会显示为”-“代替。

可以下载当前系统netstat 1.6.0版源码， grep -r -i “cmdline”。在netstat.c文件找到处理cmdline的函数代码，具体看 https://blog.51cto.com/u_7682110/4810897

重启后，由于清理了相关文件和自启动后这里的连接以及cpu高占用会自动消失。

挖矿文件处理

挖矿文件删不掉，去除i不可变属性

lsattr -R ./

chattr -i -R ./

后门处理

todeskd服务删除

systemctl list-unit-files –type=service | grep enabled

systemctl status/stop/disable todeskd

/etc/systemd/system/下找到并删除该todesk服务的systemd配置文件

system deamon-reload重新加载配置生效

shadow用户删除

伪装系统用户

date -d “1970-01-01 +19761 days” +%Y-%m-%d 发现有5个用户敏感操作在2月2日，2月8日和3月13日

同上，删除伪装的server和bash用户，tutu用户在删除时报错userdel: user tutu is currently used by process 1

隐藏的用户tutu

root密钥后门

发现刚好也是2月2日和3月13日午夜，看来这两起挖矿事件属于同一伙人
